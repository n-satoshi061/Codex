# うちの在庫ノート 要件定義書

## 1. 文書情報

- 文書名: うちの在庫ノート 要件定義書
- 対象システム: 家庭向け在庫管理アプリ
- 作成日: 2026-02-28
- 最終更新日: 2026-02-28

## 2. 背景

家庭内の食品や日用品は保管場所が分散しやすく、在庫不足や期限切れに気づきにくい。買い物前や片付け時に、必要な情報だけをすぐ確認できる仕組みが必要である。

## 3. 目的

- 家庭内在庫を品名、カテゴリ、保管場所、期限単位で管理できるようにする
- 不足品を買い物メモとして把握しやすくする
- 期限切れ在庫を通常在庫数から除外し、補充判断を誤らないようにする
- 一覧確認、在庫追加、買い物確認を役割ごとに分けて操作しやすくする

## 4. スコープ

### 4.1 対象範囲

- 在庫一覧表示
- 在庫追加
- 在庫編集
- 在庫削除
- 数量の増減
- 品名、メモ、カテゴリ、保管場所による検索
- カテゴリ絞り込み
- 買い物メモ表示
- カテゴリ、保管場所マスタ参照
- 期限切れ判定
- 品名単位の在庫集約表示

### 4.2 対象外

- ユーザー認証
- 権限管理
- 通知配信
- 購買履歴管理
- バーコード読み取り
- 複数世帯管理

## 5. 想定利用者

- 家庭内在庫を整理したい一般利用者
- 買い物前に補充対象だけ確認したい利用者
- 食品と日用品を同じ仕組みで管理したい利用者

## 6. 業務要件

### 6.1 在庫管理業務

- 利用者は在庫を品名、カテゴリ、保管場所、数量、単位、下限、期限、メモ付きで登録できること
- 利用者は同じ品名でも期限別に別在庫として登録、編集できること
- 利用者は在庫一覧を品名単位でまとめて確認できること
- 利用者は必要に応じて期限別明細を確認できること
- 利用者は一覧上で在庫の編集、削除、数量増減を行えること

### 6.2 補充判断業務

- 利用者は期限切れ在庫を通常在庫数から除外して確認できること
- 利用者は同じ品名の合計在庫数と合計下限で不足判定された品目を買い物メモとして確認できること
- 利用者は期限切れ分を含む補充判断の根拠を確認できること

## 7. 機能要件

| 要件ID | 要件名 | 内容 |
| --- | --- | --- |
| FR-01 | マスタ取得 | カテゴリと保管場所を取得し画面表示に利用できること |
| FR-02 | ダッシュボード取得 | 一覧表示用在庫、品名集約結果、買い物メモ、集計情報を取得できること |
| FR-03 | 在庫登録 | 入力フォームから在庫を登録できること |
| FR-04 | 在庫更新 | 品名、カテゴリ、保管場所、数量、単位、下限、期限、メモを更新できること |
| FR-05 | 在庫削除 | 指定在庫を削除できること |
| FR-06 | 数量更新 | 一覧から数量を 1 単位で増減できること |
| FR-07 | 検索 | 品名、メモ、カテゴリ、保管場所を対象に検索できること |
| FR-08 | 絞り込み | カテゴリ単位で一覧を絞り込めること |
| FR-09 | 品名集約表示 | 同じ品名の在庫を一覧上で 1 行にまとめて表示できること |
| FR-10 | 期限別明細表示 | 集約表示の配下で期限別在庫を確認できること |
| FR-11 | 買い物メモ表示 | 不足品を品名単位で買い物メモとして表示できること |
| FR-12 | エラー表示 | 通信失敗時に一般利用者向け文言を表示できること |

## 8. 非機能要件

### 8.1 操作性

- 在庫一覧、在庫追加、買い物メモを画面単位で切り替えて利用できること
- PC とスマートフォンの両方で利用できること
- 技術用語ではなく一般利用者向け文言を表示すること

### 8.2 性能

- 初期表示時にマスタとダッシュボード情報を取得し、通常利用で待機不能にならないこと
- 追加、編集、削除、数量更新後に再取得した結果が画面へ反映されること

### 8.3 可用性

- サーバー未応答時でも画面が異常終了せず、エラーメッセージを表示すること

### 8.4 保守性

- フロントエンドは画面表示、入力制御、API 呼び出しに責務を限定すること
- 在庫集約、期限判定、買い物メモ生成、並び替えはバックエンドで処理すること
- 主要機能に対してフロントエンド、バックエンド双方の自動テストが存在すること

## 9. 画面要件

### 9.1 在庫一覧画面

- 品名単位で集約した在庫一覧を表示すること
- 検索入力とカテゴリ絞り込みを提供すること
- 一覧上で編集、削除、数量増減を行えること
- 複数期限がある場合に期限別明細を展開できること

### 9.2 在庫追加画面

- 在庫追加フォームを表示すること
- 編集開始時は編集フォームとして同画面を利用すること
- 編集キャンセル時は一覧画面へ戻れること

### 9.3 買い物メモ画面

- 不足品のみを品名単位で一覧表示すること
- 合計在庫数、合計下限、期限切れ除外数量を確認できること

## 10. データ要件

- カテゴリはマスタ管理すること
- 保管場所はマスタ管理すること
- 在庫はカテゴリと保管場所を参照すること
- 期限切れ判定は現在日付基準で行うこと
- 期限切れ在庫は通常在庫数に含めないこと
- 初期カテゴリ、保管場所、初期在庫は seed で投入できること

## 11. 外部インターフェース要件

### 11.1 API

- `GET /api/inventory-metadata`
- `GET /api/inventory-dashboard`
- `GET /api/inventory-items`
- `POST /api/inventory-items`
- `PATCH /api/inventory-items/{inventoryItem}`
- `DELETE /api/inventory-items/{inventoryItem}`

### 11.2 利用ブラウザ

- モダンブラウザを対象とする

## 12. 制約事項

- フロントエンドは React ベースで実装する
- バックエンドは Laravel API とする
- データベースは MySQL とする
- CI/CD は GitHub Actions を利用する

## 13. 受け入れ観点

- 在庫追加画面から在庫登録ができること
- 在庫一覧画面で品名単位の集約表示ができること
- 編集ボタンから在庫を更新できること
- 削除と数量増減ができること
- 同じ品名の在庫は合計数量で不足判定されること
- 期限切れ在庫は通常在庫数から除外されること
- 買い物メモ画面で不足品が品名単位に表示されること
- 通信失敗時に一般利用者向けエラーメッセージが表示されること

# うちの在庫ノート 要件定義書

## 1. 文書情報

- 文書名: うちの在庫ノート 要件定義書
- 対象システム: 家庭向け在庫管理アプリ
- 作成日: 2026-02-28

## 2. 背景

家庭内で保有する食品や日用品は、保管場所が分散しやすく、在庫不足や期限切れに気づきにくい。買い物前に確認できる仕組みを用意し、補充漏れや重複購入を防ぎたい。

## 3. 目的

- 家庭内の在庫状況を見える化する
- 補充が必要な品目をすぐに把握できるようにする
- 期限が近い品目を早めに確認できるようにする
- カテゴリや保管場所を統一的に管理できるようにする

## 4. スコープ

### 4.1 対象範囲

- 在庫の登録
- 在庫の一覧表示
- 在庫の編集
- 在庫の検索
- カテゴリによる絞り込み
- 数量の増減
- 在庫の削除
- 買い物メモ表示
- カテゴリ、保管場所マスタの参照

### 4.2 対象外

- ユーザー認証
- 権限管理
- 通知配信
- 複数世帯の切り替え
- 購買履歴管理
- バーコード読み取り

## 5. 想定利用者

- 家庭内の在庫を整理したい利用者
- 日常の買い物前に補充対象を確認したい利用者
- 防災備蓄を含めた在庫を見える化したい利用者

## 6. 業務要件

### 6.1 在庫管理業務

- 利用者は在庫を品名、カテゴリ、保管場所、数量、単位、下限、期限、メモ付きで登録できること
- 利用者は在庫を一覧で確認できること
- 利用者は登録済み在庫のカテゴリ、保管場所、数量、下限、期限、メモを編集できること
- 利用者は残量不足の在庫を把握できること
- 利用者は期限が近い在庫を把握できること
- 利用者は不要になった在庫を削除できること

### 6.2 買い物補助業務

- 利用者は補充対象在庫を買い物メモとして確認できること
- 利用者は在庫一覧を検索やカテゴリ絞り込みで確認できること

## 7. 機能要件

| 要件ID | 要件名 | 内容 |
| --- | --- | --- |
| FR-01 | マスタ取得 | カテゴリと保管場所を取得して画面表示に利用できること |
| FR-02 | 在庫一覧取得 | 在庫一覧を取得して表示できること |
| FR-03 | 在庫登録 | 入力フォームから在庫を登録できること |
| FR-04 | 在庫更新 | 数量更新に加えて、カテゴリ、保管場所、期限、メモを含む在庫情報更新ができること |
| FR-05 | 在庫削除 | 指定在庫を削除できること |
| FR-06 | 検索 | 品名、メモ、カテゴリ、保管場所を対象に検索できること |
| FR-07 | 絞り込み | カテゴリ単位で一覧を絞り込めること |
| FR-08 | サマリー表示 | 不足件数、期限近件数、総数量を表示できること |
| FR-09 | 買い物メモ表示 | 下限以下の在庫を買い物メモとして表示できること |
| FR-10 | エラー表示 | 通信失敗時に一般ユーザー向け文言を表示できること |

## 8. 非機能要件

### 8.1 操作性

- 主要操作は 1 画面で完結すること
- PC とスマートフォンで利用できること
- 技術用語ではなく一般利用者向け文言を表示すること

### 8.2 性能

- 初期表示時に在庫一覧とマスタ取得を行い、通常利用で待機不能にならないこと
- 在庫追加、更新、削除操作の結果が画面に即時反映されること

### 8.3 可用性

- サーバー未応答時でも画面が操作不能にならず、エラーメッセージを表示すること

### 8.4 保守性

- フロントエンドはコンポーネント、フック、サービス、ユーティリティに責務分割されていること
- 主要機能に対して自動テストが存在すること

## 9. 画面要件

### 9.1 在庫管理画面

- 在庫登録フォームを表示すること
- 編集対象在庫をフォームに読み込めること
- 在庫一覧を表示すること
- サマリーを表示すること
- 買い物メモを表示すること
- 検索入力とカテゴリ絞り込みを提供すること

## 10. データ要件

- カテゴリはマスタ管理すること
- 保管場所はマスタ管理すること
- 在庫はカテゴリと保管場所を参照すること
- 初期カテゴリ、保管場所、初期在庫は seed で投入できること

## 11. 外部インターフェース要件

### 11.1 API

- `GET /api/inventory-metadata`
- `GET /api/inventory-items`
- `POST /api/inventory-items`
- `PATCH /api/inventory-items/{inventoryItem}`
- `DELETE /api/inventory-items/{inventoryItem}`

### 11.2 利用ブラウザ

- モダンブラウザを対象とする

## 12. 制約事項

- フロントエンドは React ベースで実装する
- バックエンドは Laravel API とする
- データベースは MySQL とする
- CI/CD は GitHub Actions を利用する

## 13. 受け入れ観点

- 在庫登録ができること
- 在庫一覧が取得・表示されること
- 数量増減ボタンで在庫数が更新されること
- 編集ボタンから在庫詳細を更新できること
- 削除ができること
- 下限以下の在庫が買い物メモへ表示されること
- 期限近い在庫に注意表示が出ること
- 通信失敗時に一般利用者向けのエラーメッセージが表示されること
